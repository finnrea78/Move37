All commits must adhere to the Conventional Commits Specification defined below:

.. list-table:: Conventional Commits
   :widths: 20 25 55
   :header-rows: 1

   * - Attribute
     - Format
     - Description
   * - Type
     - feat, fix, docs, style, refactor, test, chore
     - Mandatory prefix indicating the nature of the change.
   * - Scope
     - (optional, e.g., (api), (ui))
     - Optional context for the change.
   * - Subject
     - imperative tense
     - Mandatory concise description of the change.
   * - Body
     - Detailed explanation of the change
     - Optional concise description. If generated by a coding agent, add the prompt.
   * - Footer
     - Closes #XXX, BREAKING CHANGE: ...
     - Optional, for referencing issues or noting breaking changes.
